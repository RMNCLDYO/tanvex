# Tanvex

Production-ready TanStack Start template with Convex backend, Better Auth, Autumn payments, and Sentry monitoring.

For a complete, always up-to-date system view (diagrams, flows, and structure), see docs/ARCHITECTURE.md.

## Stack (Omakase)

- React 19
- Vite 7
- TanStack Start
- TanStack Router
- TanStack Query
- Convex
- Better Auth
- Autumn
- Sentry (optional)
- Tailwind CSS v4
- shadcn/ui

## What You Get

- File-based routing with TanStack Start
- Real-time data sync via Convex WebSockets
- Authentication with email and GitHub OAuth
- Payment flows with subscription management
- Production error tracking (client + server)
- Bun production server (SSR + static files on-demand)
- Dark mode with system preference detection
- Type-safe API layer (4k LOC, fully typed)

## Quick Start

```bash
# 1. Clone and install
git clone https://github.com/RMNCLDYO/tanvex.git
cd tanvex
bun install

# 2. Setup environment
cp .env.example .env.local

# 3. Initialize Convex (auto-fills deployment URLs in .env.local)
bun convex dev

# 4. Get required secrets and add to .env.local:
# - AUTUMN_SECRET_KEY from https://app.useautumn.com/sandbox/dev

# 5. Set same secrets in Convex backend
bun convex env set AUTUMN_SECRET_KEY your_key_here

# 6. Start dev server
bun run dev
```

Your app is now running at `https://localhost:3000` (HTTPS with local certificates).

### Optional Features

**GitHub OAuth:**

```bash
# Get from https://github.com/settings/developers
# Add to .env.local:
GITHUB_CLIENT_ID=your_id
GITHUB_CLIENT_SECRET=your_secret

# Set in Convex:
bun convex env set GITHUB_CLIENT_ID your_id
bun convex env set GITHUB_CLIENT_SECRET your_secret
```

**Error Tracking (Sentry):**

```bash
# Get DSN from https://sentry.io
# Add to .env.local:
SENTRY_DSN=your_dsn       # Backend

# Set in Convex:
bun convex env set SENTRY_DSN your_dsn

# Optional - for source map uploads:
SENTRY_AUTH_TOKEN=your_token
SENTRY_ORG=your_org
SENTRY_PROJECT=your_project
```

**Note:** Sentry is completely optional. The app works without it, logging to console instead.

## Environment Variables

After running `bun convex dev`, your `.env.local` will be auto-populated with Convex deployment URLs.

**Required:**

| Variable            | Get From                                      | Set In                          |
| ------------------- | --------------------------------------------- | ------------------------------- |
| `AUTUMN_SECRET_KEY` | [Autumn Dashboard](https://app.useautumn.com) | `.env.local` + Convex Dashboard |

**Optional:**

| Variable               | Get From                                                    | Set In                          |
| ---------------------- | ----------------------------------------------------------- | ------------------------------- |
| `GITHUB_CLIENT_ID`     | [GitHub OAuth Apps](https://github.com/settings/developers) | `.env.local` + Convex Dashboard |
| `GITHUB_CLIENT_SECRET` | [GitHub OAuth Apps](https://github.com/settings/developers) | `.env.local` + Convex Dashboard |
| `SENTRY_DSN`           | [Sentry Project](https://sentry.io) (backend)               | `.env.local` + Convex Dashboard |

**Auto-generated by Convex CLI:**

- `CONVEX_DEPLOYMENT` - Your deployment ID (e.g., `dev:purple-cat-123`)
- `CONVEX_URL` - Backend API URL
- `CONVEX_SITE_URL` - Public site URL

See `.env.example` for full configuration details and explanations.

## Production Server

Custom Bun server (`server.ts`) provides SSR and serves static files on-demand from `dist/client`. No extra CORS, rate limiting, preloading, ETag, or gzip are applied — rely on upstream defaults (Better Auth/Convex/Autumn/hosting).

## Monitoring

Sentry configured with:

- 10% trace sampling (production)
- 25% session replay sampling
- 100% error replay capture
- Privacy: text masking, email domain-only contexts
- Cost: Free tier ($0/month for <10k sessions/day)

## Architecture

```
src/
├── components/          # React components + shadcn/ui
├── lib/                 # Auth client, utilities
├── routes/              # File-based routes
│   ├── auth/           # Sign in/up pages
│   ├── dashboard.tsx   # Protected dashboard
│   └── pricing.tsx     # Payment flows
└── router.tsx          # Router + Sentry init

convex/
├── auth.ts             # Better Auth integration
├── autumn.ts           # Payment webhook handlers
├── customers.ts        # User data queries
├── dashboard.ts        # Dashboard queries
└── schema.ts           # Database schema

server.ts               # Production server (Bun)
```
